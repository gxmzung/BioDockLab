# BioDockLab 교수/학과장 검토 패키지

## 1. 프로젝트 한 줄 정의

BioDockLab은 의료진을 대체하는 AI 진단 시스템이 아니라, 환자·간호사·의사·약사·원무·연구자·보안관리자가 각자 필요한 의료·바이오 정보만 안전하게 볼 수 있도록 설계한 역할 기반 의료·바이오 데이터 설명 및 보안 감사 플랫폼 프로토타입입니다.

## 2. 검토 요청 목적

이 프로젝트는 현재 학부생 수준의 웹 프로토타입입니다.

컴퓨터공학 관점에서는 다음 기능을 구현했습니다.

- 역할 기반 로그인
- 역할별 화면 분리
- 환자용 설명 리포트
- 간호사용 바이탈·인수인계 보조 화면
- 의사용 결과 요약 화면
- 약사용 처방량·약물상호작용 검토 화면
- 원무용 예약·동의서·서류 상태 화면
- 연구자용 단백질-화합물 가상 도킹 실험 화면
- 보안관리자용 접근 로그 및 위험 이벤트 화면
- 플랫폼 관리자용 권한 매트릭스 화면
- 권한 없는 접근 차단
- 접근 로그 기록
- 역할별 접근 정책 JSON 분리

하지만 실제 의료 현장 관점에서는 다음 부분을 검토받아야 합니다.

- 병원 업무흐름과 맞는 역할 구분인지
- 간호사 화면의 정보 구조가 타당한지
- 환자에게 보여주는 설명 문구가 위험하지 않은지
- EMR/간호기록/처방 시스템과 혼동되지 않는지
- 개인정보보호와 보안 측면에서 부족한 점은 무엇인지
- 실제 병원 또는 간호학과 연구 주제로 발전 가능한지

## 3. 핵심 주장

BioDockLab이 주장하는 것은 다음입니다.

1. 의료·바이오 데이터는 사용자 역할별로 다르게 보여야 한다.
2. 환자는 어려운 전문용어보다 이해 가능한 설명이 필요하다.
3. 간호사·의사·약사·원무·연구자는 서로 다른 정보 범위가 필요하다.
4. 권한 없는 접근은 차단되어야 하고, 접근 기록은 남아야 한다.
5. 연구자는 실명 환자 데이터가 아니라 비식별 연구 데이터와 실험 로그를 다뤄야 한다.
6. 바이오그래머는 생명과학 문제를 소프트웨어 시스템으로 풀어내는 새로운 융합형 직무가 될 수 있다.

## 4. 주장하지 않는 것

BioDockLab은 다음을 주장하지 않습니다.

- AI가 진단한다.
- AI가 처방한다.
- 의사나 약사를 대체한다.
- 간호기록이나 EMR을 대체한다.
- 실제 환자 치료에 바로 적용 가능하다.
- 실제 신약 개발을 완료했다.
- 임상적으로 검증된 치료 추천 시스템이다.

## 5. 현재 구현 화면

| 화면 | 목적 |
|---|---|
| Login | 역할 기반 진입 |
| Patient View | 환자용 쉬운 설명 리포트 |
| Nurse Workspace | 바이탈·인수인계·투약 전 확인 보조 |
| Doctor Workspace | 의사용 결과 요약 및 설명 리포트 |
| Pharmacist Workspace | 처방량·상호작용·유전자 적합성 검토 |
| Administration View | 예약·동의서·서류 상태, 민감정보 마스킹 |
| Research Lab | 단백질-화합물 가상 도킹 실험 |
| Security Center | 접근 로그, 권한 차단 이벤트 |
| Admin Console | 역할별 권한 매트릭스, 정책 확인 |
| My Workspace | 역할별 주요 업무, 접근 가능 데이터, 제한 데이터 |

## 6. 교수님께 검토받고 싶은 핵심 질문

1. 병원 현장 관점에서 역할 구분이 타당한가?
2. 간호사 화면에 들어간 바이탈, 인수인계, 투약 전 확인 정보가 적절한가?
3. 환자용 설명 리포트에서 위험하거나 과장된 표현은 없는가?
4. 약사 화면에서 처방량·상호작용·유전자 적합성을 다루는 방식이 적절한가?
5. 원무 화면에서 어느 정보까지 보여주는 것이 현실적인가?
6. 연구자 화면의 단백질-화합물 도킹 실험이 보건·간호·생명과학 연구 관점에서 의미가 있는가?
7. 보안센터의 접근 로그·내부자 과다열람 탐지 개념이 현장 문제와 연결되는가?
8. 이 프로젝트가 취창업동아리/뉴스/교내 발표에서 어떤 표현으로 소개되는 것이 안전한가?

## 7. 검토 방식 제안

- 10~15분 화면 시연
- 문서 기반 검토
- 간호사 화면 중심 피드백
- 의료 표현 위험도 검토
- 개인정보/보안 관점 피드백
- 추후 창업동아리 또는 산학협력 방향 자문

## 8. 현재 단계

현재 BioDockLab은 병원에 바로 납품할 제품이 아니라, 의료·바이오 데이터 접근 방식과 바이오그래머 직무 가능성을 보여주는 검토용 프로토타입입니다.

교수님 검토 이후 다음 단계는 다음과 같습니다.

1. 잘못된 의료 표현 수정
2. 간호사·의사·약사·원무 화면 현실성 개선
3. 보안·개인정보 문서 보강
4. UI 고급화
5. 교내 창업지원단 또는 산학협력단 상담
6. 병원/학과 협력 가능성 검토
