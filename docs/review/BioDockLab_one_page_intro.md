# BioDockLab 한 장 소개서

## 한 줄 정의

BioDockLab은 의료진을 대체하는 AI 진단 시스템이 아니라, 환자·간호사·의사·약사·원무·연구자·보안관리자가 각자 필요한 정보만 안전하게 볼 수 있도록 설계한 역할 기반 의료·바이오 데이터 설명 및 보안 감사 플랫폼 프로토타입입니다.

## 왜 필요한가

의료·바이오 데이터는 복잡하고 민감합니다.

같은 환자 데이터라도 사용자마다 필요한 정보가 다릅니다.

- 환자는 쉬운 설명과 상담 준비 정보가 필요합니다.
- 간호사는 바이탈, 인수인계, 투약 전 확인 정보가 필요합니다.
- 의사는 환자 상태와 결과 요약, 상담용 리포트가 필요합니다.
- 약사는 처방량, 약물상호작용, 유전자 적합성 정보가 필요합니다.
- 원무는 예약, 동의서, 서류 상태만 필요합니다.
- 연구자는 비식별 연구 데이터와 실험 로그가 필요합니다.
- 보안관리자는 누가 어떤 정보에 접근했는지 확인해야 합니다.

## 현재 구현된 화면

- 로그인/회원가입
- 환자 화면
- 간호사 화면
- 의사 화면
- 약사 화면
- 원무 화면
- 연구자 화면
- 보안센터
- 플랫폼 관리자 콘솔
- 역할별 워크스페이스
- 권한 매트릭스
- 접근 차단 화면
- 감사 로그

## 현재 구현된 보안 개념

- 역할 기반 접근제어
- 민감정보 마스킹
- 접근 로그 자동 기록
- 권한 없는 접근 차단
- 내부자 과다열람 의심 이벤트
- 플랫폼 관리자 권한 매트릭스
- 보안 문서화

## 주의할 점

BioDockLab은 현재 학부생 프로토타입입니다.

아직 실제 의료기관 시스템, EMR, 간호기록, 처방 시스템과 연동되어 있지 않습니다.

상용화를 위해서는 의료현장 검토, 개인정보보호 검토, 보안 아키텍처 검토, 법률 검토, 의료 표현 검수가 필요합니다.
