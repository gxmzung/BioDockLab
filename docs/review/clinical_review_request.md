# BioDockLab 현장 검토 요청서

## 검토 요청 목적

BioDockLab은 컴퓨터공학 전공 학생이 개발한 의료·바이오 데이터 역할 기반 접근 플랫폼 프로토타입입니다.

현재 구현은 기술적 가능성을 보여주는 수준이며, 실제 병원 현장 업무흐름과 의료정보 표현이 타당한지는 전문가 검토가 필요합니다.

## 검토받고 싶은 질문

### 1. 역할 구분

현재 역할은 다음과 같습니다.

- 환자
- 간호사
- 의사
- 약사
- 원무
- 연구자
- 보안관리자
- 플랫폼 관리자

질문:

- 이 역할 구분이 병원 현장 관점에서 타당한가?
- 간호사 역할이 따로 필요한가?
- 간호사와 의사의 정보 접근 범위는 어떻게 달라야 하는가?
- 원무가 보면 안 되는 정보는 무엇인가?
- 약사가 반드시 봐야 하는 정보는 무엇인가?

### 2. 간호사 화면

현재 간호사 화면 가정:

- 바이탈 요약
- 인수인계 요약
- 투약 전 확인
- 환자 상태 변화 알림
- 동의 상태 확인
- 민감정보 최소 표시

질문:

- 간호사 화면에 반드시 들어가야 할 정보는 무엇인가?
- 넣으면 위험하거나 부적절한 정보는 무엇인가?
- 인수인계 보조 화면은 어떤 구조가 현실적인가?
- 간호기록/EMR과 구분해야 할 점은 무엇인가?

### 3. 환자 설명 화면

질문:

- 환자에게 보여주는 설명 문구가 의료적으로 위험하거나 과장되어 보이는가?
- “예상 효능”, “권장도”, “위험도” 같은 표현이 부적절한가?
- 어떤 표현을 피해야 하는가?
- 어떤 문구를 반드시 넣어야 하는가?

### 4. 보안/개인정보

질문:

- 역할별 접근제어 방향이 타당한가?
- 접근 로그와 내부자 과다열람 탐지 개념이 현장에서 의미가 있는가?
- 실제 병원에서 더 중요한 보안 이슈는 무엇인가?
- 환자 동의와 연구 데이터 활용을 어떻게 분리해야 하는가?

### 5. 연구자 화면

질문:

- 단백질-화합물 가상 도킹 실험 화면이 대학 연구자에게 설명자료로 의미가 있는가?
- 연구자 화면에 어떤 샘플 데이터가 더 필요할까?
- 생명공학/간호/보건 연구 관점에서 보완할 점은 무엇인가?

## 검토 방식 제안

- 10~15분 데모 시연
- 화면 캡처 자료 검토
- 문서 기반 피드백
- 간호사 역할 화면 중심 피드백
- 의료 표현 위험도 검토
