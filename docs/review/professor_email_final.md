# 교수님께 보낼 메일 최종안

제목: 의료·바이오 데이터 역할 기반 플랫폼 프로토타입 검토 요청드립니다

안녕하세요, 교수님.

저는 배재대학교 컴퓨터공학과 이영준입니다.

현재 BioDockLab이라는 의료·바이오 데이터 기반 웹 프로토타입을 개발하고 있습니다. 이 프로젝트는 AI가 진단하거나 처방하는 시스템이 아니라, 환자·간호사·의사·약사·원무·연구자·보안관리자처럼 역할에 따라 필요한 정보만 다르게 보여주고, 민감정보 접근을 제한하며, 접근 기록을 남기는 방식의 역할 기반 플랫폼입니다.

현재 구현된 주요 화면은 다음과 같습니다.

- 환자용 검사결과 설명 화면
- 간호사용 바이탈·인수인계·투약 전 확인 보조 화면
- 의사용 결과 요약 리포트 화면
- 약사용 처방량·약물상호작용 검토 화면
- 원무용 예약·동의서·서류 상태 화면
- 연구자용 단백질-화합물 가상 도킹 실험 화면
- 보안관리자용 접근 로그·권한 차단 이벤트 화면
- 플랫폼 관리자용 역할별 권한 매트릭스 화면

다만 저는 컴퓨터공학 전공이라 실제 병원 현장의 업무 흐름, 간호사 관점에서 필요한 정보, EMR/간호기록 시스템과 구분해야 할 점, 환자에게 보여주는 의료정보 표현에서 주의해야 할 부분을 충분히 검증하지 못했습니다.

그래서 교수님께 다음 부분에 대해 짧게라도 검토 또는 조언을 부탁드리고 싶습니다.

1. 현재 역할 구분이 병원 현장 관점에서 타당한지
2. 간호사 화면에 어떤 정보가 반드시 들어가야 하는지
3. 간호기록이나 EMR을 대체하는 것처럼 보이지 않으려면 어떤 점을 조심해야 하는지
4. 환자용 설명 문구가 의료적으로 과장되거나 위험해 보이지 않는지
5. 개인정보·보안 측면에서 학생 프로토타입이 주의해야 할 점은 무엇인지

BioDockLab은 현재 상용 서비스가 아니라 학부생 수준의 프로토타입입니다. 다만 장기적으로는 의료 현장과 협력하여 의료정보를 더 안전하고 이해하기 쉽게 전달하는 시스템으로 발전시켜보고 싶습니다.

가능하시다면 10~15분 정도 화면을 보여드리고 피드백을 받을 수 있을지 여쭙고 싶습니다.
