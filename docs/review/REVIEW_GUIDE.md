# Review Guide

## Purpose

This document explains how BioDockLab should be reviewed.

## Recommended Review Questions

### Product

- Is the problem definition clear?
- Is the target user realistic?
- Is the MVP scope too broad?
- Does the workflow match real research or medical-adjacent work?

### Bio / Medical Boundary

- Are any claims too strong?
- Are clinical assumptions safe?
- Is the project clearly separated from diagnosis or treatment?
- Is expert review needed before the next step?

### Software

- Is the system structure understandable?
- Are the modules separated clearly?
- Is the API direction reasonable?
- Is the documentation enough for future contributors?

### Ethics and Data

- Is sensitive data avoided?
- Is anonymization policy clear?
- Are limitations visible?
- Is user role access reasonable?

## Current Review Status

BioDockLab is currently suitable for concept review, prototype review, and software architecture review.

It is not suitable for clinical validation review yet.
