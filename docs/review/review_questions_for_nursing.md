# 간호학과/간호사 관점 검토 질문지

## 1. 간호사 역할 화면 검토

현재 Nurse Workspace는 다음 정보를 보여줍니다.

- 환자 상태 요약
- 바이탈 요약
- 투약 전 확인
- 인수인계 요약
- 동의 상태
- 민감정보 최소 표시
- 접근 로그 기록

### 질문

1. 간호사 화면에 바이탈 요약을 넣는 것이 적절한가?
2. 바이탈 항목은 혈압, 맥박, 체온, SpO₂ 정도면 충분한가?
3. 간호사가 환자용 설명 리포트를 볼 수 있어야 하는가?
4. 인수인계 요약 자동화가 현장에서 도움이 될 가능성이 있는가?
5. 인수인계 요약에 들어가면 안 되는 정보는 무엇인가?
6. 투약 전 확인 화면은 약사 화면과 어떻게 구분되어야 하는가?
7. 간호사가 약물상호작용 정보를 어느 수준까지 보는 것이 적절한가?
8. 간호기록/EMR 대체처럼 보이지 않으려면 어떤 문구가 필요할까?
9. 간호사 화면에서 가장 먼저 보여야 할 정보는 무엇인가?
10. 간호사 업무 흐름상 이 화면이 실제로 쓰일 수 있는 순간은 언제인가?

## 2. 환자 설명 화면 검토

현재 Patient View는 다음 정보를 보여줍니다.

- 환자 요약 정보
- 검사결과 쉬운 설명
- 후보 치료제 비교
- 상담 준비 문구
- 의사에게 물어볼 질문

### 질문

1. 환자에게 “예상 효능”, “권장도”, “위험도” 같은 표현을 보여줘도 되는가?
2. “긍정적인 치료 반응이 기대됩니다” 같은 표현은 위험한가?
3. 환자용 설명에서 반드시 들어가야 할 경계 문구는 무엇인가?
4. 환자가 오해하지 않도록 어떤 문장을 피해야 하는가?
5. 환자에게 보여주는 수치 정보는 어느 수준까지 허용될 수 있는가?
6. 환자용 리포트는 의사 상담 전 자료로 의미가 있는가?

## 3. 간호사-의사-약사 업무 분리 검토

### 질문

1. 의사 화면과 간호사 화면의 가장 큰 차이는 무엇이어야 하는가?
2. 간호사 화면과 약사 화면의 가장 큰 차이는 무엇이어야 하는가?
3. 원무 화면과 간호사 화면에서 겹치면 안 되는 정보는 무엇인가?
4. 투약 전 확인은 간호사와 약사가 각각 어떻게 다르게 다뤄야 하는가?
5. 환자 상태 변화 알림은 어느 직군에게 먼저 보여야 하는가?

## 4. 보안/개인정보 관점 검토

### 질문

1. 병원에서 실제로 내부자 과다열람 문제가 중요한가?
2. 간호사·의사·약사·원무의 접근 로그를 모두 남기는 것이 현실적인가?
3. 접근 사유 입력이 필요한 화면은 어떤 화면인가?
4. 환자 동의 상태는 어느 직군까지 볼 수 있어야 하는가?
5. 연구자가 비식별 데이터만 접근한다는 구조가 적절한가?
6. 학생 프로토타입이 가장 조심해야 할 개인정보 이슈는 무엇인가?

## 5. 학과/연구 관점 질문

1. 간호학과 또는 보건의료 연구 주제로 확장 가능성이 있는가?
2. 간호사 인수인계 보조, 환자 설명 보조, 보안 감사 중 어느 방향이 가장 현실적인가?
3. 캡스톤디자인 또는 창업동아리 프로젝트로 발전시키려면 무엇을 줄이고 무엇을 강화해야 하는가?
4. 실제 병원 관계자에게 보여주기 전에 반드시 고쳐야 할 부분은 무엇인가?
