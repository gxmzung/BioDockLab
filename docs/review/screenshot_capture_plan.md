# BioDockLab 화면 캡처 계획

## 목적

교수님, 학과장, 동아리 심사, 뉴스/홍보 자료에 사용할 화면 캡처 순서를 정리합니다.

화면 캡처는 코드를 보여주기보다 “무엇을 구현했는지”와 “무엇을 검토받고 싶은지”를 보여주는 목적입니다.

## 캡처 1. Login Screen

### 보여줄 것

- 역할 기반 로그인
- 환자/간호사/의사/약사/원무/연구자/보안관리자 역할
- 플랫폼 관리자 계정은 버튼이 아니라 별도 로그인

### 설명

BioDockLab은 처음부터 사용자 역할에 따라 접근 범위를 분리합니다.

## 캡처 2. Patient View

### 보여줄 것

- 환자 요약
- 쉬운 설명 리포트
- 상담 준비 문구

### 설명

환자는 진단을 받는 것이 아니라, 의료진 상담을 준비하기 위한 쉬운 설명을 받습니다.

## 캡처 3. Nurse Workspace

### 보여줄 것

- 바이탈 요약
- 인수인계 요약
- 투약 전 확인
- 간호기록 대체 아님 경고 문구

### 설명

간호사 화면은 간호기록/EMR을 대체하지 않고, 인수인계와 환자 상태 확인을 보조하는 화면입니다.

## 캡처 4. Doctor Workspace

### 보여줄 것

- 의사용 결과 요약
- 분자적 근거
- 예측 점수
- 환자 설명 리포트

### 설명

의사는 환자 설명과 전문가용 해석을 함께 확인합니다.

## 캡처 5. Pharmacist Workspace

### 보여줄 것

- 처방량
- 약물상호작용
- 유전자 적합성
- 약사 검토 메모

### 설명

약사는 처방의 안전성과 상호작용을 검토하는 역할입니다.

## 캡처 6. Administration View

### 보여줄 것

- 예약
- 동의서
- 서류 상태
- 진단/처방 상세 마스킹

### 설명

원무는 업무에 필요한 최소 정보만 확인합니다.

## 캡처 7. Research Lab

### 보여줄 것

- 단백질 타겟
- 후보 화합물
- 도킹 결과표
- 실험 파라미터
- 3D 구조 화면

### 설명

연구자는 비식별 연구 데이터와 가상 도킹 실험 흐름을 봅니다.

## 캡처 8. Security Center

### 보여줄 것

- 접근 로그
- 권한 없는 접근 이벤트
- 위험 이벤트
- 권한 정책

### 설명

모든 접근은 기록되고, 권한 밖 접근은 차단됩니다.

## 캡처 9. Platform Admin Console

### 보여줄 것

- 역할 요약
- 실시간 권한 매트릭스
- 정책 위험 체크
- 상용화 보안 체크리스트

### 설명

플랫폼 관리자는 역할과 권한 정책을 관리합니다.

## 캡처 10. My Workspace

### 추천 역할

- 간호사
- Bio Security Architect
- Virtual Lab Developer

### 설명

BioDockLab은 사용자 역할뿐 아니라 바이오그래머 직무 역할까지 설명합니다.
