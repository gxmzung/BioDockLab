# 바이오그래머 기반 BioDockLab 프로젝트 계획서

## 1. 프로젝트 배경

생명과학은 더 이상 실험실 안에서만 끝나는 분야가 아니다.  
의료 데이터, 인공지능, 유전자 편집, 단백질 구조 예측, 디지털 트윈, 보안, 원격 상담, 연구 데이터 플랫폼이 서로 연결되고 있다.

문제는 기술이 발전할수록 정보가 더 복잡해진다는 점이다.

환자는 “내 몸에 어떤 의미인지”를 알고 싶어 한다.  
의사는 진료와 상담 관점에서 정보를 본다.  
간호사는 바이탈, 인수인계, 투약 전 확인이 중요하다.  
약사는 처방량, 약물상호작용, 복약지도가 중요하다.  
원무는 예약, 동의서, 서류, 보험/청구 흐름이 중요하다.  
연구자는 비식별 데이터, 단백질, 변이, 실험 로그가 중요하다.  
보안관리자는 누가 어떤 정보에 접근했는지를 봐야 한다.

같은 의료·바이오 데이터라도 사용자마다 필요한 설명과 접근 권한이 다르다.

---

## 2. 바이오그래머 정의

바이오그래머는 생명과학 문제를 소프트웨어 시스템으로 풀어내는 사람이다.

단순히 바이오와 코딩을 합친 사람이 아니라, 복잡한 생명과학·의료·바이오 데이터를 실제 사람이 이해하고 사용할 수 있는 시스템으로 바꾸는 직업이다.

한 줄 정의:

> 바이오그래머는 복잡한 생명과학 기술을 사람에게 전달 가능한 소프트웨어 시스템으로 바꾸는 사람이다.

---

## 3. 바이오그래머가 필요한 이유

의료와 바이오 기술은 빠르게 발전하고 있다.

- 유전자 편집
- 단백질 구조 예측
- 개인 건강 데이터
- 의료 빅데이터
- 디지털 트윈
- AI 분석
- 원격 상담
- 의료정보 보안

하지만 기술이 발전할수록 일반 사용자가 이해하기는 더 어려워진다.

따라서 앞으로는 기술 자체를 개발하는 사람뿐 아니라, 기술을 사람에게 맞게 번역하고, 역할별로 보여주고, 안전하게 관리하는 사람이 필요하다.

이 역할을 바이오그래머라고 정의한다.

## 5. BioDockLab 플랫폼 개요

BioDockLab은 바이오그래머가 어떤 시스템을 만들 수 있는지 보여주는 웹 기반 프로토타입이다.

BioDockLab은 AI가 진단하거나 처방하는 시스템이 아니다.  
의료진을 대체하지 않는다.  
EMR이나 간호기록을 대체하지 않는다.

BioDockLab은 역할에 따라 필요한 정보만 다르게 보여주고, 민감정보 접근을 제한하며, 접근 기록을 남기는 역할 기반 의료·바이오 데이터 설명 및 보안 감사 플랫폼이다.

## 7. 핵심 기능

### 7.1 역할 기반 접근제어

사용자 역할에 따라 볼 수 있는 정보가 다르다.

- 환자는 자신의 설명 리포트만 확인
- 간호사는 바이탈과 인수인계 중심 확인
- 약사는 처방·상호작용 중심 확인
- 원무는 예약·동의서·서류 상태 중심 확인
- 연구자는 비식별 연구 데이터만 확인
- 보안관리자는 접근 로그 확인

### 7.2 환자 설명 보조

환자가 검사결과나 의료 정보를 이해하기 쉽게 정리한다.

주의:

- 진단하지 않음
- 처방하지 않음
- 치료 추천하지 않음
- 의료진 상담 전 참고자료로 제한

### 7.3 간호사 업무 보조

간호사 화면은 다음 정보를 중심으로 구성한다.

- 바이탈 요약
- 투약 전 확인
- 인수인계 요약
- 동의 상태
- 민감정보 최소 표시

### 7.4 연구·교육 기능

연구자와 학생을 위한 기능:

- Protein Atlas
- 단백질 구조 정보
- 변이·질환·리간드 연결
- Virtual Docking Lab
- 교육·연구용 단백질 도감

### 7.5 보안 감사 로그

모든 민감정보 접근은 기록된다.

- 누가 접근했는가
- 언제 접근했는가
- 어떤 정보에 접근했는가
- 권한 없는 접근이 있었는가
- 내부자 과다열람 의심 이벤트가 있는가

## 8. 확장 기능

### 8.1 BioDockLab Case Match

익명 유사 사례 기반 상담 준비 기능.

목표:

- 비슷한 증상/검사결과 사례 참고
- 의사에게 물어볼 질문 정리
- 상담 전 정보 준비

주의:

- 유사 사례로 진단하지 않음
- 치료법 추천하지 않음
- 최종 판단은 의료진이 수행

### 8.2 BioDockLab Care Connect

사전문진 기반 온라인 상담 매칭 기능.

목표:

- 환자가 병원 방문 전 증상과 정보를 정리
- 응급 위험 신호가 있으면 병원 방문 안내
- 응급성이 낮은 경우 온라인 상담 연결
- 의사가 상담 전에 요약 정보를 확인

주의:

- 자가진단 아님
- AI 진단 아님
- 의료진 상담 연결 보조 기능

## 16. 최종 메시지

BioDockLab은 바이오그래머라는 새로운 직업이 실제로 어떤 시스템을 만들 수 있는지 보여주는 구현 사례다.

이 플랫폼은 의료진을 대체하지 않는다.  
진단하거나 처방하지 않는다.  
대신 의료·바이오 데이터를 역할별로 나누고, 사람이 이해할 수 있게 설명하며, 접근권한과 보안 로그를 관리한다.

BioDockLab의 목표는 복잡한 생명과학 기술을 사람에게 전달 가능한 시스템으로 바꾸는 것이다.
