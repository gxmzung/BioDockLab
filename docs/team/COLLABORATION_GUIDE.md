# Collaboration Guide

## Purpose

This document defines how collaborators should work on BioDockLab.

## Collaboration Principles

- Keep claims conservative.
- Document assumptions.
- Separate code, research notes, and business documents.
- Avoid real sensitive medical data.
- Ask for expert review when bio-domain claims are involved.
- Keep GitHub commits readable.

## Contribution Flow

Recommended workflow:

1. Define the issue or feature.
2. Check whether it belongs to code, research, business, or documentation.
3. Update the relevant module.
4. Document assumptions and limitations.
5. Commit with a clear message.
6. Review before presentation or external sharing.

## Documentation Rule

Every important feature should have one of the following:

- Architecture note
- Research note
- Validation note
- Limitation note
- Roadmap update

## Communication Rule

BioDockLab should be explained as a research software prototype, not a clinical or medical product.
