# BioDockLab Initial Team Candidate Mapping

## 1. Purpose

이 문서는 BioDockLab을 개인 프로토타입에서 대학동아리 기반 MVP 팀으로 확장하기 위한 초기 팀 후보자 역할 배치를 정리한다.

주의:

- 이 문서는 내부 검토용이다.
- 외부 공개 README, 발표자료, 제안서에는 실명 대신 역할명 또는 이니셜을 사용한다.
- 실제 환자 데이터, 병원 내부 데이터, 의료기관 비공개 데이터는 개인 인맥으로 수집하지 않는다.
- 의료 데이터 활용은 공개 데이터, 샘플 데이터, 비식별 데이터, 공식 연구협력 절차를 기준으로 한다.

## 5. Immediate Recruiting Message

BioDockLab은 의료·바이오 데이터를 역할별로 안전하게 보여주고, 환자 설명, 연구자 실험, 보안 감사 로그를 연결하는 웹 기반 프로토타입이다.

현재는 실제 환자 데이터나 실제 진단 기능을 사용하지 않고, 샘플 데이터 기반 MVP로 개발한다.

팀원에게 기대하는 것은 단순한 노동이 아니라 다음과 같은 결과물이다.

- GitHub 기여 기록
- 포트폴리오 산출물
- 발표/공모전 경험
- 의료·바이오 융합 프로젝트 경험
- 창업/사업화 검토 경험
- 활동비 기반 프로젝트 참여

## 6. Data Safety Rule

BioDockLab은 의료 데이터를 다루는 프로젝트처럼 보일 수 있으므로, 다음 원칙을 지킨다.

1. 실제 환자 데이터 사용 금지
2. 병원 내부 데이터 무단 수집 금지
3. 공개 데이터 또는 샘플 데이터 우선 사용
4. 비식별화 원칙 준수
5. 의료진/교수 자문 전 진단·처방 표현 금지
6. AI 기능은 설명과 상담 준비 보조로 제한
7. 실제 상용화 전 법률·개인정보·보안 검토 필수
