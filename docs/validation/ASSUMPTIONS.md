# Assumptions

BioDockLab is built on several early-stage assumptions.

## Product Assumptions

- Researchers need better experiment data organization.
- Structured experiment records can improve comparison and reporting.
- AI-assisted summaries can reduce documentation burden.
- Simulation can help explore experimental conditions.

## Technical Assumptions

- FastAPI is sufficient for the MVP backend.
- Rule-based analysis is safer than unvalidated ML at the current stage.
- Sample data can be used for workflow testing.
- Modular folder structure will help long-term development.

## Research Assumptions

- Organoid and CFPS workflows can be modeled as structured experiment records.
- Digital twin-style simulation can become useful after validation.
- Expert feedback is required before any serious bio-domain claim.

## Risk

These assumptions must be tested through expert review, real data, and controlled validation.
