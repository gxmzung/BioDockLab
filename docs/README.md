# BioDockLab Documentation

This directory contains the full documentation structure for BioDockLab.

## Main Sections

- api: API specifications and boundaries
- architecture: system context, data flow, module map, and technical structure
- business: business direction and commercialization boundary
- decisions: architecture decision records
- ethics: ethical boundary and AI claim policy
- evidence: award, presentation, and validation evidence
- operations: setup, developer guide, and deployment notes
- presentation: presentation flow and Q&A preparation
- research: research expansion directions
- review: external review materials
- roadmap: implementation and commercialization roadmap
- security: data protection and security boundary
- team: team roles and collaboration guide
- validation: validation status, assumptions, and limitations

## Documentation Principle

BioDockLab documentation should make three things clear:

1. What the project currently does.
2. What the project does not claim.
3. What must be validated before real-world use.

BioDockLab is a research software prototype, not a validated medical or clinical system.
