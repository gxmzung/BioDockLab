# Data Protection

## Purpose

This document defines the data protection direction for BioDockLab.

## Current Data Policy

BioDockLab should use only non-sensitive data at the MVP stage.

Allowed data:

- Sample experiment data
- Mock clinical-style data
- Anonymized demonstration records
- Synthetic test data

Not allowed at the current stage:

- Real patient identifiers
- Real clinical records
- Sensitive personal information
- Hospital production data

## Future Requirements

Before handling real sensitive data, BioDockLab would need:

- Expert review
- Legal review
- Security architecture review
- Access control
- Audit logging
- Data retention policy
- Consent policy

## Current Position

BioDockLab is not ready for real sensitive data handling.
