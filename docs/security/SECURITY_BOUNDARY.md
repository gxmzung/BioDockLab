# Security Boundary

## Purpose

This document defines the security boundary of BioDockLab.

BioDockLab is a research software prototype and should not be treated as a production medical or clinical system.

## In Scope

Security planning may include:

- Access policy design
- Role-based access planning
- Audit log concept
- Sensitive data boundary
- Sample data safety
- Future authentication planning

## Out of Scope

BioDockLab currently does not support:

- Production medical data handling
- Real patient record storage
- Certified clinical security workflow
- Hospital-grade compliance
- Production authentication and authorization

## Data Handling Principle

BioDockLab should only use:

- Sample data
- Anonymized demonstration data
- Non-sensitive research workflow data

Real patient data or sensitive personal data should not be used at the current stage.

## Current Status

Security is currently in planning and documentation stage.
