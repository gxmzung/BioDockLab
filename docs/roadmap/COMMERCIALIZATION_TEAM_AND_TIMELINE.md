# BioDockLab Commercialization Team and Timeline

## 1. Purpose

이 문서는 BioDockLab이 학부생 프로토타입에서 팀 MVP, 파일럿 제안, 상용화 검토 단계로 가기 위해 필요한 개발 기간, 팀 구성, 역할별 요구 스킬과 도구를 정리한다.

BioDockLab은 단순 웹앱이 아니라 다음 요소가 결합된 플랫폼이다.

- 역할 기반 의료·바이오 데이터 접근제어
- 환자 설명 보조
- 간호사·의사·약사·원무 업무 보조
- 연구자용 Protein Atlas / Virtual Docking Lab
- 보안 감사 로그
- 향후 Care Connect / Case Match 확장

따라서 상용화까지는 단순 개발 기간만이 아니라 의료 자문, 보안 검토, 개인정보 검토, 법률 검토, 사용자 검증이 필요하다.

## 2. Development Timeline

## 2.1 Current Stage — Review Prototype

예상 기간: 현재 완료 단계

현재 상태:

- 역할별 UI 프로토타입 구현
- 샘플 데이터 기반 시연 가능
- 교수님 검토 요청 자료 작성
- GitHub 문서화 및 Project 생성
- Phase 1~8 일정표 작성

가능한 수준:

- 교수/의사/간호사/약사/보안 전문가 검토
- 발표 및 시연
- 창업 아이디어 검증
- 팀원 모집용 포트폴리오 자료

불가능한 수준:

- 실제 환자 데이터 사용
- 실제 의료 판단
- 실제 온라인 진료 운영
- 병원 EMR/HIS 연동
- 보험/청구 연동

---

## 2.2 Team MVP Stage

예상 기간: 3~6개월

목표:

- 팀 기반 v1.0 Review MVP 완성
- 역할별 화면 안정화
- 백엔드 권한제어 강화
- 감사 로그 DB화
- Protein Atlas 초기 구현
- 의료 표현 위험도 제거

필요 인력:

- Product Architect 1명
- Frontend Developer 1명
- Backend / Security Developer 1명
- Clinical Workflow Advisor 1명
- Bio / Research Advisor 1명
- QA / Documentation 담당 1명

대학동아리 단위 가능 여부:

- 가능
- 단, 실제 환자 데이터 없이 샘플 데이터 기반으로 진행해야 함
- 참여자에게 포트폴리오, GitHub 기여, 발표 경험, 연구/창업 경험을 제공할 수 있음

## 2.3 Pilot-Ready Demo Stage

예상 기간: 6~12개월

목표:

- 학과, 창업지원단, 병원 관계자에게 제안 가능한 데모 패키지 완성
- 보안·개인정보 문서화
- 전문가 피드백 반영
- 데모 영상 및 발표자료 완성
- 제한적 교육/연구용 파일럿 제안 가능

필요 인력:

- Product Lead 1명
- Frontend Developer 1~2명
- Backend Developer 1명
- Security Engineer 1명
- Bio Data Curator 1명
- Clinical Advisor 1~2명
- QA / Compliance 1명
- Design / Presentation 담당 1명

대학동아리 단위 가능 여부:

- 대부분 가능
- 단, 의료법·개인정보·원격진료 검토는 외부 전문가 자문 필요
- 실제 병원 파일럿은 학교, 교수, 병원 관계자 협력이 필요함

---

## 2.4 Commercial Feasibility Stage

예상 기간: 12~18개월

목표:

- 상용화 가능성 검토
- 실제 사용자 인터뷰
- 의료기관/연구기관/교육기관 대상 제안
- 법률·개인정보·보안 검토
- 서비스 구조와 수익모델 검증

가능한 초기 상용화 방향:

1. 교육·연구용 Protein Atlas / Virtual Lab
2. 환자 상담 준비 리포트
3. 병원/클리닉용 역할별 업무 보조 SaaS
4. 의료·바이오 데이터 접근 감사 로그 플랫폼

주의:

- Care Connect, Case Match, Telehealth Matching은 규제 검토 전 실제 운영 금지
- 진단/처방/치료 추천 기능으로 홍보하면 안 됨

---

## 2.5 Regulated Commercial Service Stage

예상 기간: 18~36개월 이상

목표:

- 실제 서비스 출시
- 보안 인증, 개인정보 처리방침, 법률 검토
- 병원 또는 의료기관 연동 검토
- 실제 환자 데이터 처리 가능성 검토
- 운영팀 구축

필요 인력:

- CEO / Product Lead
- CTO
- Clinical Lead
- Frontend Team
- Backend Team
- Security / Compliance Team
- AI / Data Team
- Bioinformatics Team
- DevOps Team
- QA / Validation Team
- Legal / Privacy Advisor
- Business Development

대학동아리 단위 가능 여부:

- 불가능
- 법인, 투자, 전문가 자문, 보안 체계, 법률 검토가 필요함
- 대학동아리 단계에서는 여기까지 직접 운영하지 않고, 파일럿 제안과 검증 자료 확보가 목표임

## 3. Recommended Commercialization Order

가장 현실적인 순서는 다음과 같다.

| Step | Product Direction | Difficulty | Regulation Risk | Recommended Timing |
|---:|---|---|---|---|
| 1 | Education / Research Protein Atlas | Medium | Low | First |
| 2 | Virtual Docking Lab Demo | Medium | Low-Medium | First |
| 3 | Patient Explanation Report | Medium | Medium | After advisor review |
| 4 | Role-Based Clinical Workspace | High | Medium-High | After clinical review |
| 5 | Security Audit Log Platform | High | Medium-High | After backend/security hardening |
| 6 | Case Match | Very High | High | Concept only at first |
| 7 | Care Connect / Telehealth Matching | Very High | Very High | Legal/medical review required |
| 8 | EMR/HIS Integration | Very High | Very High | Last stage |

## 5.3 Activity-Based Team Operation

BioDockLab의 초기 팀 운영은 정규 고용 형태가 아니라 **대학동아리 기반 프로젝트팀**으로 시작한다.

이유:

- 초기 단계에서 상용 스타트업 수준의 인건비를 지급하기 어렵다.
- 아직 실제 매출이 없는 검토용 프로토타입 단계다.
- 팀원에게는 활동비와 포트폴리오 가치를 제공하는 방식이 현실적이다.
- 공모전, 창업지원금, 학과 지원금, 동아리 지원금, 외부 후원금을 활용할 수 있다.

운영 원칙:

| 항목 | 운영 방식 |
|---|---|
| 기본 보상 | 활동비, 식비, 교통비, 회의비 |
| 성과 보상 | 공모전 수상금 또는 지원금 일부 배분 |
| 포트폴리오 보상 | GitHub 기여, 역할 명시, 발표자료 참여 |
| 경험 보상 | 의료·바이오 융합 프로젝트 경험, 전문가 피드백 경험 |
| 책임 범위 | 실제 의료 서비스 운영이 아니라 샘플 데이터 기반 MVP 개발 |
| 법적 경계 | 실제 환자 데이터, 진단, 처방, 원격진료 운영은 제외 |

초기 활동비 지급 기준 예시:

| 역할 | 활동비 기준 |
|---|---|
| 핵심 개발자 | 월 단위 또는 마일스톤 단위 활동비 |
| 자문 팀원 | 회의/검토 횟수 기준 활동비 |
| 문서/디자인 담당 | 산출물 기준 활동비 |
| 발표/공모전 참여자 | 프로젝트 기여도 기준 보상 |
| 외부 전문가 | 가능하면 자문비 또는 식사/교통비 제공 |

주의:

BioDockLab 팀은 “무료로 사람을 쓰는 팀”이 아니라, 초기 자금 제약 안에서 활동비와 포트폴리오 가치를 제공하는 대학 기반 프로젝트팀으로 운영한다.

## 6. Recruiting Priority

## 6.1 First Recruiting Priority

1. Backend / Security Developer
2. Clinical Workflow Advisor
3. Bio / Research Advisor

Reason:

- 현재 UI 프로토타입은 존재함
- 다음 약점은 백엔드 권한, 의료현장 검토, 바이오 데이터 검증임

## 6.2 Second Recruiting Priority

1. Frontend Developer
2. AI/Data Engineer
3. QA/Documentation

Reason:

- UI 완성도와 데이터 구조화 필요
- 설명 요약과 유사 사례 기능은 기획 검증 후 구현해야 함

## 6.3 Third Recruiting Priority

1. Legal / Privacy Advisor
2. Business Development
3. Hospital / Startup Support Connector

Reason:

- 상용화 또는 파일럿 제안 단계에서 필요함
- 초기 프로토타입 단계에서는 외부 자문 형태가 현실적임

## 7. Realistic Conclusion

BioDockLab은 혼자서 프로토타입까지는 만들 수 있다.

하지만 상용화까지는 팀이 필요하다.

현실적인 구분은 다음과 같다.

| Stage | Solo Possible | Team Required |
|---|---|---|
| UI prototype | Yes | No |
| Professor review package | Yes | No |
| GitHub documentation | Yes | No |
| Role-based MVP | Partial | Yes |
| Backend RBAC hardening | Partial | Yes |
| Protein Atlas MVP | Partial | Yes |
| Security audit platform | No | Yes |
| Real patient data | No | Yes |
| Telehealth matching | No | Yes |
| EMR/HIS integration | No | Yes |
| Commercial service | No | Yes |

Final position:

BioDockLab should first grow as a **university club-based role-based healthcare/bio data MVP project**, then move toward **education/research pilot**, and only after expert/legal/security review should it be considered for real healthcare commercialization.
