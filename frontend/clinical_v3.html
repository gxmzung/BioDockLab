역할 기반 의료·바이오 데이터 접근 플랫폼

환자, 의사, 약사, 원무, 연구자, 보안관리자가 각자 필요한 정보만 접근하도록 분리합니다. 모든 접근은 감사 로그로 기록되며, 권한 밖 접근은 차단됩니다.

역할 기반 접근제어 환자·의사·약사·원무·연구자·보안 역할별 화면 분리
민감정보 마스킹 원무·연구자 화면에서 진단/처방/개인식별 정보 제한
보안 감사 로그 누가, 언제, 어떤 화면에 접근했는지 자동 기록
연구/설명 보조 실제 진단·처방이 아닌 연구와 설명 보조 목적
내부 직무 분리 Data, AI, 설명, 보안설계, Virtual Lab 직무는 관리자 문서에서 별도 관리
데모 계정은 아래 버튼으로 바로 로그인할 수 있습니다. 실제 상용화 버전에서는 병원/기관 인증, 2단계 인증, 관리자 승인이 필요합니다. 플랫폼 관리자 계정은 보안을 위해 버튼으로 노출하지 않고 이메일/비밀번호로만 로그인합니다.
환자 외 역할은 실제 서비스에서는 면허, 소속기관, 관리자 승인 절차가 필요합니다. 현재는 발표용 프로토타입이므로 데모 계정으로 즉시 생성됩니다. 플랫폼 관리자와 내부 바이오그래머 직무 계정은 관리자 문서에서 별도로 관리합니다.
